### Understanding Protected Routes

Protected routes are routes that require authentication to access. In Expo Router, we protect routes by:

1. **Grouping routes** with `(parentheses)` folders — one `(auth)` group, one `(app)` group
2. **Guarding** each group with `<Stack.Protected guard={...}>` and a boolean condition
3. Expo Router **redirects automatically** when a guard becomes false — no manual `<Redirect>` or `router.replace()` needed

### Organize routes with groups

- [ ] Reorganize your `app/` folder to separate auth and protected routes for the Spacecraft app:


**🔭 Hint:** Route groups (folders with parentheses) don't affect the URL. `(auth)/index.tsx` maps to `/`, not `/(auth)/`.

### Create an Auth Context

- [ ] Create `src/context/AuthContext.tsx` to manage authentication state:

```javascript
// src/context/AuthContext.tsx
import { createContext, useContext, useState, ReactNode } from "react";

type AuthContextType = {
  user: boolean;
  signIn: () => void;
  signOut: () => void;
};

const AuthContext = createContext<AuthContextType | null>(null);

export function AuthProvider({ children }: { children: ReactNode }) {
  const [user, setUser] = useState(false);

  const signIn = () => setUser(true);
  const signOut = () => setUser(false);

  return (
    <AuthContext.Provider value={{ user, signIn, signOut }}>
      {children}
    </AuthContext.Provider>
  );
}

export function useAuth() {
  const context = useContext(AuthContext);
  if (!context) {
    throw new Error("useAuth must be used within an AuthProvider");
  }
  return context;
}
```

### Setup the root layout with the protected navigator

The cleanest way to protect routes is `Stack.Protected`: wrap each group in `<Stack.Protected guard={...}>`. When a `guard` is `false` the group becomes unreachable and Expo Router redirects automatically — no `<Redirect>` or `router.replace()` anywhere.

- [ ] Update `app/_layout.tsx` to wrap the app with the providers **and** the guarded navigator:

```javascript
// app/_layout.tsx
import { Stack } from "expo-router";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { PaperProvider } from "react-native-paper";
import { AuthProvider, useAuth } from "../src/context/AuthContext";
import { theme } from "../src/theme/theme";

const queryClient = new QueryClient();

function RootNavigator() {
  const { user } = useAuth();

  return (
    <Stack screenOptions={{ headerShown: false }}>
      {/* 🔓 Reachable only when signed OUT */}
      <Stack.Protected guard={!user}>
        <Stack.Screen name="(auth)" />
      </Stack.Protected>

      {/* 🔐 Reachable only when signed IN */}
      <Stack.Protected guard={user}>
        <Stack.Screen name="(app)" />
      </Stack.Protected>
    </Stack>
  );
}

export default function RootLayout() {
  return (
    <QueryClientProvider client={queryClient}>
      <PaperProvider theme={theme}>
        <AuthProvider>
          <RootNavigator />
        </AuthProvider>
      </PaperProvider>
    </QueryClientProvider>
  );
}
```

**🔭 Hint:** `RootNavigator` must live inside `<AuthProvider>` so it can read `user` with the `useAuth` hook.

### Create the protected route layout

The guard now lives in the root navigator, so `(app)/_layout.tsx` only has to render its screens:

- [ ] Create `app/(app)/_layout.tsx`:

```javascript
// app/(app)/_layout.tsx
import { Stack } from "expo-router";

export default function AppLayout() {
  return <Stack screenOptions={{ headerShown: false }} />;
}
```

**🔐 How it works:**

1. The `<Stack.Protected guard={user}>` in `app/_layout.tsx` wraps the whole `(app)` group
2. When `user` is falsy, `(app)` is unreachable and the router falls back to `(auth)` → the login screen
3. When `user` becomes truthy, the guard flips and Expo Router navigates into `(app)` automatically

### Create the auth layout

- [ ] Create `app/(auth)/_layout.tsx` for the authentication screens:

```javascript
// app/(auth)/_layout.tsx
import { Stack } from "expo-router";

export default function AuthLayout() {
  return (
    <Stack screenOptions={{ headerShown: false }}>
      <Stack.Screen name="index" />
      <Stack.Screen name="terms" />
    </Stack>
  );
}
```

No redirect here either: when `user` becomes truthy, the `guard={!user}` on the `(auth)` group turns the whole group off and the router moves into `(app)`.

```javascript
import { useAuth } from "../../src/context/AuthContext";

export default function LoginScreen() {

    <View>

      <Button title="Sign in" onPress={signIn} />
    </View>
  );
```


**🔭 Hint:** No `router.replace()` needed — flipping `user` flips the guards, and Expo Router handles the redirect for you.
