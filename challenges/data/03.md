# Add an `Offline` component

📱 Works only on a smartphone.

## 📡 What you will learn

- Read the network state with `@react-native-community/netinfo`
- Add an offline feedback message

## 👾 Before we start the exercise

- [ ] We are going to install and use `@react-native-community/netinfo`
- [ ] Have a look at [`@react-native-community/netinfo` documentation](https://github.com/react-native-netinfo/react-native-netinfo)

## 👨‍🚀 Exercise 3

- [ ] Install the library

```console
npx expo install @react-native-community/netinfo
```

NetInfo is now very simple: you read the network state with a single hook (or function) — no Context provider or extra components needed.

- [ ] Get the network state with the `useNetInfo()` hook:

```javascript
import { useNetInfo } from "@react-native-community/netinfo";

function Offline() {
  const { isConnected } = useNetInfo();

  if (isConnected === false) {
    return <Text>⚠️ You are offline</Text>;
  }

  return null;
}
```

**🔭 Hint:** You can also read it once with the `NetInfo.fetch()` function:

```javascript
import NetInfo from "@react-native-community/netinfo";

const state = await NetInfo.fetch();
console.log("Is connected?", state.isConnected);
```

- [ ] Create an `Offline.tsx` component that displays a message when the user is not connected.
- [ ] Render `<Offline />` on top of your screens so the message shows everywhere.

**🔭 Hint:** Test with `isConnected === false`, not `!isConnected`: `isConnected` is `null` while NetInfo resolves the first time, and `!null` is `true` (false positive). To preview the UI while online, temporarily flip the condition.
